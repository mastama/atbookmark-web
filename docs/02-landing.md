# atbookmark — Landing Page UI

Refer to `initial-plan.md` for:
- Product vision
- Design system
- Tech stack
- UI principles

You are a Senior Frontend Engineer and Product Designer.

Your task is to design and implement the LANDING PAGE ONLY
for the atbookmark web application.

--------------------------------------------------
SCOPE LIMITATION
--------------------------------------------------
- Frontend UI only
- No backend logic
- No authentication logic
- Use mock data only
- Do NOT implement dashboard or internal pages

--------------------------------------------------
PAGE GOALS
--------------------------------------------------
- Explain the value of atbookmark in < 5 seconds
- Encourage users to register
- Establish a playful & bright brand identity

--------------------------------------------------
SECTIONS TO IMPLEMENT
--------------------------------------------------

1. Hero Section
- Headline:
  "Save now. Read later. No more tab chaos."
- Subheadline:
  "Turn messy browser tabs into a clean personal library."
- Primary CTA:
  "Get Started — Free"
- Secondary CTA:
  "Login"
- Visual:
  - Abstract illustration
  - Floating bookmark cards animation

--------------------------------------------------

2. Pain Point Section
- Show the problem:
  - Too many open tabs
  - High RAM usage
  - Lost important articles
- Visual metaphor:
  - Overheated laptop
  - Browser warning

--------------------------------------------------

3. Feature Highlights
Display 3–4 features:
- Save bookmarks instantly
- Organize with folders & tags
- Clean reader mode
- Access from anywhere

Use:
- Large icons
- Short copy
- Bright accent colors

--------------------------------------------------

4. How It Works
Steps:
1. Save a link
2. Organize automatically
3. Read when ready

--------------------------------------------------

5. Social Proof (Mock)
- Short testimonial cards
- Friendly avatars
- Playful tone

--------------------------------------------------

6. Final CTA
- Headline:
  "Start building your reading library today."
- Button:
  "Create Free Account"

--------------------------------------------------
UX & DESIGN NOTES
--------------------------------------------------
- Bright color usage
- Smooth scroll
- Framer Motion animations
- Skeleton loaders not required
- Responsive (desktop-first)

--------------------------------------------------
DELIVERABLES
--------------------------------------------------
- Page layout
- Section components
- Tailwind classes
- Motion effects
- Clean, readable code
